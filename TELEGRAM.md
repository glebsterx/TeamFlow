# TeamFlow — Telegram Integration

## Команды бота

| Команда | Описание |
|---------|---------|
| `/task` | Создание задачи через FSM диалог |
| `/my` | Мои задачи |
| `/week` | Недельная доска задач |
| `/meeting` | Фиксация результатов встречи |
| `/meetings` | История встреч за 30 дней |
| `/digest` | Еженедельный дайджест |
| `/overdue` | Просроченные задачи |
| `/start`, `/menu` | Главное меню с inline кнопками |
| `/help` | Справка |
| `/cancel` | Отменить текущий диалог (FSM) |

### Требования
- Privacy mode отключён (для работы в группах)
- Inline клавиатуры включены

---

## FSM: Создание задачи (/task)

```
WAIT_TITLE    → пользователь вводит название
WAIT_ASSIGNEE → вводит @username или пропускает
WAIT_DUE_DATE → вводит дату или пропускает
WAIT_DEFINITION → вводит критерий готовности
CONFIRM       → показывается карточка + кнопки [Создать] [Отменить]
```

FSM хранится в Redis (aiogram MemoryStorage → RedisStorage).

---

## Callback Data Protocol

Формат: `entity:entity_id:action`

### Task Actions
```
task:{id}:start     → статус DOING
task:{id}:done      → статус DONE
task:{id}:block     → статус BLOCKED (запрашивает причину)
```

### Message Candidate (автопарсинг)
```
candidate:confirm:{message_id}  → создать задачу
candidate:reject:{message_id}   → игнорировать
```

## Автоматический парсинг сообщений

Бот анализирует обычные сообщения в чате и предлагает создать задачу.

### Критерии (confidence score)
- Глагол действия: "сделай", "проверь", "реализуй", "исправь" (30+ слов)
- Упоминание @username → assignee
- Дата: "завтра", "в пятницу", "через 3 дня", "к понедельнику"
- Минимум 5 слов в сообщении

### Flow
```
Сообщение → MessageParsingService → confidence >= 0.6?
  → да: подтверждение пользователю → [Да] / [Нет]
  → нет: игнорируется
```

### Паттерны поручений
```
"@john нужно проверить API до пятницы"
"сделай ... до ..."
"нужно ... к ..."
"берёшь ...?"
```
