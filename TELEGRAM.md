# TeamFlow — Telegram Integration

## Команды бота

| Команда | Описание |
|---------|---------|
| `/task` | Создание задачи через FSM диалог (или ответом на сообщение) |
| `/tasks` | Список задач с фильтрами (статус, проект, приоритет) |
| `/my` | Мои активные задачи, сгруппированы по статусу и приоритету |
| `/sprint` | Текущий спринт с inline-кнопками смены статуса |
| `/week` | Недельная доска задач |
| `/meeting` | Фиксация результатов встречи |
| `/meetings` | История встреч за 30 дней |
| `/digest` | Еженедельный дайджест |
| `/overdue` | Просроченные задачи |
| `/remind <id> <30m\|2h\|1d\|18:00>` | Напоминание о задаче |
| `/start`, `/menu` | Главное меню с inline кнопками |
| `/help` | Справка по всем командам |
| `/cancel` | Отменить текущий диалог (FSM) |

### Требования
- Privacy mode отключён (для работы в группах)
- FSM хранится в **MemoryStorage** (Redis убран)

---

## FSM: Создание задачи (/task)

```
waiting_for_title       → пользователь вводит название
waiting_for_description → вводит описание или пропускает
waiting_for_due_date    → вводит дедлайн (ДД.ММ ЧЧ:ММ) или пропускает
```

Дедлайны принимаются в **московском времени (UTC+3)**, хранятся в UTC.

### /task в ответ на сообщение
Если `/task` написан как reply на сообщение в чате — задача создаётся автоматически из текста сообщения:
- title = текст сообщения (первые 255 символов)
- description = "Из сообщения от @username:\n\n..."
- source = CHAT_MESSAGE

## Callback Data Protocol

Формат: `entity:entity_id:action`

### Task Actions (из созданной задачи)
```
task:{id}:start   → статус DOING
task:{id}:done    → статус DONE
task:{id}:block   → статус BLOCKED
```

### Sprint Board (из /sprint)
```
sp_s:{task_id}:{new_status}:{sprint_id}  → смена статуса задачи
sp_r:{sprint_id}                         → обновить доску спринта
```

### Main Menu
```
menu:task     → запустить /task FSM
menu:my       → показать мои задачи
menu:sprint   → показать текущий спринт
menu:week     → показать /week
menu:digest   → показать /digest
menu:meeting  → запустить /meeting FSM
menu:meetings → история встреч
menu:help     → справка
```

## /sprint — интерактивная доска

Команда `/sprint` показывает текущий активный спринт:
- Прогресс-бар (▰▱ X/Y задач)
- Список задач с inline-кнопками: **▶ Взять** / **✓ Готово** / **↩ Вернуть**
- Кнопка ↗ #{id} — ссылка на задачу в Web UI
- Кнопка 🔄 Обновить

Подкоманды:
```
/sprint list        — список всех спринтов со статусами
/sprint start <id>  — запустить спринт (деактивирует текущий)
/sprint end <id>    — завершить спринт
```

---

## /my — персональная доска

Показывает активные задачи текущего пользователя (по `assignee_telegram_id` или `@username`).
Сгруппировано: 🔄 В работе → 📝 К выполнению → 🚫 Заблокировано → ⏸ Отложено.
Внутри каждой группы — сортировка по приоритету (URGENT→HIGH→NORMAL→LOW).

---

## /remind — напоминания

```
/remind 42 30m    — напомнить о задаче #42 через 30 минут
/remind 42 2h     — через 2 часа
/remind 42 1d     — через 1 день
/remind 42 18:00  — сегодня в 18:00 МСК
```

## Автоматический парсинг сообщений

Бот анализирует обычные сообщения в чате.

### Критерии
- Глагол действия: "сделай", "проверь", "реализуй", "исправь"
- Упоминание @username → assignee
- `#НазваниеПроекта` → привязка к проекту
- Минимум 5 слов в сообщении

### Уведомления о дедлайнах (deadline_notifier.py)
- Проверка каждые 30 минут
- Уведомления за **24 часа** и **3 часа** до дедлайна
- Защита от дублей через таблицу `deadline_notifications`
- Настраивается: `DEADLINE_NOTIFY_HOURS=24,3` в `.env`

## Прокси

Бот поддерживает работу через прокси (если Telegram заблокирован):
- **SOCKS5:** `TELEGRAM_PROXY_URL=socks5://user:pass@host:port`
- **HTTP/HTTPS:** `TELEGRAM_PROXY_URL=http://user:pass@host:port`

> MTProxy **не поддерживается** — был попробован в v0.8.15 (sidecar `teamflow-mtg`) и не заработал, удалён.

Настраивается через `backend/.env` или через UI настроек (⚙️ → Прокси).

## Telegram Mini App

При наличии `WEBAPP_URL` в `.env` в главном меню появляется кнопка **🌐 Открыть TeamFlow**.
Mini App показывает персональную доску задач и сводку активного спринта прямо в Telegram.
URL: `{WEBAPP_URL}/app` (требует HTTPS для работы через кнопку в боте).

## Telegram Login Widget (авторизация в веб-интерфейсе)

Веб-интерфейс поддерживает авторизацию через Telegram Login Widget (кнопка «Войти через Telegram»).

### Настройка в BotFather

Перед использованием нужно привязать домен к боту:

1. Открыть [@BotFather](https://t.me/BotFather)
2. `/mybots` → выбрать бота → **Bot Settings** → **Domain**
3. Указать домен веб-интерфейса (например: `teamflow.example.com`)

> Без этой настройки Telegram не будет рендерить виджет на внешнем домене.

### Требования

- **HTTPS** — виджет работает только на HTTPS (или localhost)
- Домен должен быть указан в BotFather
- `TELEGRAM_BOT_USERNAME` должен быть задан в `backend/.env`

### Как работает

1. Пользователь нажимает кнопку «Log in with Telegram» на странице `/login`
2. Telegram открывает окно авторизации
3. После подтверждения — данные отправляются на `POST /api/auth/telegram`
4. Backend проверяет подпись (HMAC-SHA256) и создаёт JWT-токены
5. `telegram_id` сохраняется в `teamflow_my_user_id` — Dashboard автоматически определяет текущего пользователя
